B. Cutover sequence

Deploy dual-write

Enable scheduler.v2 in shadow mode — write both paths, read legacy.

Both paths agree on 100 sample reservations

Backfill

Replay historical rows into the event store; rebuild viewstore. Verify row counts match.

Viewstore count == legacy count; zero parity diffs

Flip reads

Point queries at the viewstore. Legacy still written as a safety net.

24h soak in STE with no error-rate regression

Decommission

Stop legacy writes; archive slot_reservation; remove the flag.

Rollback at any gate
Flip scheduler.v2 back to off → reads return to the legacy table instantly. Backfill is additive, so no destructive change exists until the final decommission step. Do not drop the legacy table until 7 days post-flip.
