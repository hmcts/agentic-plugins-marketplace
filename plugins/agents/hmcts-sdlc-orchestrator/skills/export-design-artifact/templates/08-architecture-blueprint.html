§ 02 · Notes

Data ownership

The scheduler context owns reservation state end-to-end. The write side never reads from the viewstore; the read side is rebuilt purely by replaying the event store. Only public.scheduler.* events cross the boundary — private events stay internal per subscriptions-descriptor.yaml.
