# OpenSpec Skill Bundle

Four skills that cover the full [OpenSpec](https://github.com/Fission-AI/OpenSpec) spec-driven change workflow:

| Skill | What it does |
|---|---|
| `openspec-explore` | Thinking-partner mode. No code, just investigation, diagrams, and clarifying questions before a change is proposed. |
| `openspec-propose` | Create a new change and generate all required artifacts (proposal, design, tasks) in one step. |
| `openspec-apply-change` | Implement tasks from a change, ticking each task complete as you go until done or blocked. |
| `openspec-archive-change` | Finalise a completed change — check artifacts and tasks, sync delta specs, move to `openspec/changes/archive/`. |

These skills were generated by the OpenSpec CLI and are bundled here for marketplace installation. Attribution for the underlying workflow and skill content belongs to the OpenSpec project.

## Prerequisite — the `openspec` CLI

Every skill in this plugin shells out to the `openspec` binary for `openspec list`, `openspec status`, `openspec instructions`, `openspec new change`, etc. You must have the CLI installed and on your `PATH` before the skills will work.

Install the CLI before enabling this plugin — see the [installation instructions](https://github.com/Fission-AI/OpenSpec#installation) in the upstream repository. If the binary is not on your `PATH`, skills will load but fail with `command not found` at the first shell-out.

## Usage

Each skill auto-triggers on its intent description. Examples:

```
# Explore
Let's think through what it would take to add audit logging.

# Propose
Propose a change: switch the job runner from Celery to RQ.

# Apply
Start implementing the add-audit-logging change.
Continue working through the tasks.

# Archive
Archive the add-audit-logging change — it's finished.
```

The typical flow is: `explore` → `propose` → `apply-change` → `archive-change`. Any skill can be invoked independently — they read the current state of `openspec/` to figure out where you are.

## Installation

```
/plugin install openspec@agentic-plugins-marketplace
```

Then ensure the `openspec` CLI is on your `PATH`.

## Why bundled

All four skills read and write under the same `openspec/` directory and compose into a single workflow. Bundling means one install and one version to keep aligned with the upstream CLI.
